1. Formulario Simple - Custom handleSubmit

¿Qué es un Formulario Simple con Custom handleSubmit? Un formulario simple en React usa el evento onSubmit del <form> para manejar el envío. Creamos una función handleSubmit personalizada para procesar los datos, prevenir el comportamiento por defecto (recarga de página), y realizar acciones como validar o enviar datos.

Ejemplo Práctico Vamos a crear un formulario básico en AddTask.jsx para agregar una tarea. Usamos useState para manejar el valor del input.

Modifica src/pages/AddTask.jsx: 

import { useState } from 'react';
import { useNavigate } from 'react-router-dom';

function AddTask() {
  const [taskText, setTaskText] = useState('');
  const navigate = useNavigate();

  const handleSubmit = (e) => {
    e.preventDefault(); // Previene la recarga de la página
    if (taskText.trim()) {
      console.log('Tarea agregada:', taskText); // Simulamos el envío
      setTaskText(''); // Limpia el input
      navigate('/tasks'); // Redirige a la lista de tareas
    } else {
      console.log('El campo no puede estar vacío');
    }
  };

  return (
    <div className="container mx-auto p-4 max-w-md">
      <h1 className="text-3xl font-bold mb-6 text-center">Agregar Nueva Tarea</h1>
      <form onSubmit={handleSubmit} className="space-y-4">
        <div>
          <label htmlFor="task" className="block text-lg font-medium mb-2">
            Tarea:
          </label>
          <input
            type="text"
            id="task"
            value={taskText}
            onChange={(e) => setTaskText(e.target.value)}
            className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
            placeholder="Escribe una tarea"
          />
        </div>
        <button
          type="submit"
          className="w-full py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-200"
        >
          Agregar Tarea
        </button>
      </form>
    </div>
  );
}

export default AddTask;

Explicación Detallada:

• useState maneja el estado del input (taskText).

• handleSubmit es una función personalizada que:

o Previene el comportamiento por defecto del formulario con e.preventDefault().

o Verifica si el campo no está vacío.

o Simula el envío (en consola) y limpia el input.

o Redirige al usuario a /tasks usando useNavigate.

• El formulario usa onSubmit para llamar a handleSubmit.

• Estilizado con TailwindCSS para responsividad: el contenedor max-w-md asegura que sea mobile-friendly, y las clases como w-full hacen que los elementos se adapten a la pantalla.

Prueba:

• Ve a /add-task.

• Escribe una tarea y envía el formulario. Verifica en la consola que se registre y que seas redirigido.

• Si el campo está vacío, no se envía y se muestra un mensaje en consola.

2. Implementación de SweetAlert2

¿Qué es SweetAlert2? SweetAlert2 es una librería para mostrar alertas modales personalizadas, hermosas y responsivas. Se integra fácilmente con React para mostrar confirmaciones, errores o éxitos después de acciones como enviar un formulario.

Ejemplo Práctico Agregamos SweetAlert2 para mostrar una alerta de éxito al agregar una tarea.

Primero, importa la librería en AddTask.jsx: 

import Swal from 'sweetalert2';

Modifica handleSubmit en src/pages/AddTask.jsx: 

const handleSubmit = (e) => {
  e.preventDefault();
  if (taskText.trim()) {
    console.log('Tarea agregada:', taskText);
    setTaskText('');
    Swal.fire({
      title: '¡Éxito!',
      text: 'Tarea agregada correctamente.',
      icon: 'success',
      confirmButtonText: 'OK',
      timer: 3000, // Cierra automáticamente después de 3 segundos
      timerProgressBar: true,
      showCloseButton: true,
      customClass: {
        popup: 'swal-popup-custom', // Para estilizado personalizado
      },
    }).then(() => {
      navigate('/tasks');
    });
  } else {
    Swal.fire({
      title: 'Error',
      text: 'El campo no puede estar vacío.',
      icon: 'error',
      confirmButtonText: 'OK',
    });
  }
};

Explicación Detallada:

• Swal.fire configura la alerta:

o Para éxito: Muestra un título, texto, icono de éxito, botón de confirmación, temporizador y barra de progreso.

o Para error: Muestra un alerta de error si el campo está vacío.

• La alerta es responsiva por defecto en SweetAlert2, adaptándose a móviles.

• Usamos then para redirigir después de cerrar la alerta.

• Puedes agregar CSS personalizado para swal-popup-custom en index.css para más estilizado.

Prueba:

• Envía el formulario con una tarea: Ve la alerta de éxito y redirección.

• Envía vacío: Ve la alerta de error.

3. action y formData

¿Qué es action y formData? En React, action en un formulario puede ser una función que maneja el envío usando FormData para recolectar los datos del formulario automáticamente. Esto es útil para formularios complejos sin manejar cada input manualmente.

Ejemplo Práctico Usamos action como una función que procesa FormData.

Modifica src/pages/AddTask.jsx para usar action: 

import { useState } from 'react';
import { useNavigate } from 'react-router-dom';
import Swal from 'sweetalert2';

function AddTask() {
  const navigate = useNavigate();

  const formAction = (formData) => {
    const taskText = formData.get('task'); // Obtiene el valor del input por name
    if (taskText.trim()) {
      console.log('Tarea agregada:', taskText);
      Swal.fire({
        title: '¡Éxito!',
        text: 'Tarea agregada correctamente.',
        icon: 'success',
        confirmButtonText: 'OK',
      }).then(() => {
        navigate('/tasks');
      });
    } else {
      Swal.fire({
        title: 'Error',
        text: 'El campo no puede estar vacío.',
        icon: 'error',
        confirmButtonText: 'OK',
      });
    }
  };

  return (
    <div className="container mx-auto p-4 max-w-md">
      <h1 className="text-3xl font-bold mb-6 text-center">Agregar Nueva Tarea</h1>
      <form action={formAction} className="space-y-4">
        <div>
          <label htmlFor="task" className="block text-lg font-medium mb-2">
            Tarea:
          </label>
          <input
            type="text"
            id="task"
            name="task" // Name es necesario para FormData
            className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
            placeholder="Escribe una tarea"
          />
        </div>
        <button
          type="submit"
          className="w-full py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-200"
        >
          Agregar Tarea
        </button>
      </form>
    </div>
  );
}

export default AddTask;

Explicación Detallada:

• action es una función que recibe formData (un objeto FormData).

• Usamos formData.get('task') para obtener el valor del input por su name.

• Esto simplifica el manejo de formularios grandes, ya que no necesitas useState para cada campo.

• Integramos SweetAlert2 para feedback.

Prueba:

• Envía el formulario: FormData recolecta el dato automáticamente.

6. Checkbox Dinámicos

¿Qué es Checkbox Dinámicos? Checkbox dinámicos se generan a partir de una lista (e.g., categorías), manejando un array de valores seleccionados con useState.

Ejemplo Práctico Agregamos checkboxes dinámicos para categorías.

Modifica src/pages/AddTask.jsx: 

// ... imports ...

function AddTask() {
  const [urgent, setUrgent] = useState(false);
  const [categories, setCategories] = useState([]);
  const navigate = useNavigate();

  const categoryOptions = ['Trabajo', 'Personal', 'Ocio'];

  const handleCategoryChange = (e) => {
    const value = e.target.value;
    if (e.target.checked) {
      setCategories([...categories, value]);
    } else {
      setCategories(categories.filter((cat) => cat !== value));
    }
  };

  const formAction = (formData) => {
    const taskText = formData.get('task');
    const description = formData.get('description');
    const priority = formData.get('priority');
    if (taskText.trim() && description.trim()) {
      console.log('Tarea agregada:', { taskText, description, priority, urgent, categories });
      Swal.fire({
        title: '¡Éxito!',
        text: 'Tarea agregada correctamente.',
        icon: 'success',
        confirmButtonText: 'OK',
      }).then(() => {
        navigate('/tasks');
      });
    } else {
      Swal.fire({
        title: 'Error',
        text: 'Los campos tarea y descripción no pueden estar vacíos.',
        icon: 'error',
        confirmButtonText: 'OK',
      });
    }
  };

  return (
    <div className="container mx-auto p-4 max-w-md">
      <h1 className="text-3xl font-bold mb-6 text-center">Agregar Nueva Tarea</h1>
      <form action={formAction} className="space-y-4">
        // ... campos anteriores ...
        <div>
          <label className="block text-lg font-medium mb-2">Categorías:</label>
          {categoryOptions.map((cat) => (
            <div key={cat} className="flex items-center">
              <input
                type="checkbox"
                id={cat}
                value={cat}
                checked={categories.includes(cat)}
                onChange={handleCategoryChange}
                className="mr-2"
              />
              <label htmlFor={cat}>{cat}</label>
            </div>
          ))}
        </div>
        <button
          type="submit"
          className="w-full py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-200"
        >
          Agregar Tarea
        </button>
      </form>
    </div>
  );
}

export default AddTask;

Explicación Detallada:

• useState maneja un array categories.

• handleCategoryChange agrega/remueve valores del array.

• Los checkboxes se generan dinámicamente con map sobre categoryOptions.

• En formAction, incluimos categories en el log.

Prueba:

• Selecciona categorías y envía: Verifica en consola el array seleccionado.

8. Validaciones Asincronas

¿Qué es Validaciones Asincronas? Validaciones asincronas involucran operaciones como llamadas a API (e.g., verificar si una tarea ya existe).

Ejemplo Práctico Agregamos una validación asincrona para verificar si la tarea ya existe (simulada con Promise).

Modifica formAction en src/pages/AddTask.jsx: 

const formAction = async (formData) => {
  const taskText = formData.get('task');
  const description = formData.get('description');
  const priority = formData.get('priority');
  if (!taskText.trim() || !description.trim()) {
    Swal.fire({ title: 'Error', text: 'Campos requeridos', icon: 'error' });
    return;
  }

  // Validación asincrona simulada
  try {
    const exists = await new Promise((resolve) => {
      setTimeout(() => resolve(taskText === 'Tarea Existente'), 1000);
    });
    if (exists) {
      Swal.fire({ title: 'Error', text: 'Tarea ya existe', icon: 'error' });
      return;
    }

    console.log('Tarea agregada:', { taskText, description, priority, urgent, categories });
    Swal.fire({
      title: '¡Éxito!',
      text: 'Tarea agregada correctamente.',
      icon: 'success',
      confirmButtonText: 'OK',
    }).then(() => {
      navigate('/tasks');
    });
  } catch (error) {
    Swal.fire({ title: 'Error', text: 'Error en la validación', icon: 'error' });
  }
};

Explicación Detallada:

• Usamos async/await para simular una llamada API.

• Si la tarea "existe", mostramos error.

• Esto demuestra validaciones asincronas sin bloquear la UI.

9. Hook Formik (useFormik)

¿Qué es Hook Formik (useFormik)? Formik es una librería para manejar formularios en React. useFormik es un hook que facilita el manejo de estado, validaciones y envío.

Ejemplo Práctico Cambiamos el formulario a Formik.

Modifica src/pages/AddTask.jsx: 

import { useFormik } from 'formik';
import { useNavigate } from 'react-router-dom';
import Swal from 'sweetalert2';

function AddTask() {
  const navigate = useNavigate();

  const formik = useFormik({
    initialValues: {
      task: '',
      description: '',
      priority: 'media',
      urgent: false,
      categories: [],
    },
    onSubmit: (values) => {
      console.log('Tarea agregada:', values);
      Swal.fire({
        title: '¡Éxito!',
        text: 'Tarea agregada correctamente.',
        icon: 'success',
      }).then(() => {
        navigate('/tasks');
      });
    },
  });

  const categoryOptions = ['Trabajo', 'Personal', 'Ocio'];

  const handleCategoryChange = (value) => {
    const categories = formik.values.categories.includes(value)
      ? formik.values.categories.filter((cat) => cat !== value)
      : [...formik.values.categories, value];
    formik.setFieldValue('categories', categories);
  };

  return (
    <div className="container mx-auto p-4 max-w-md">
      <h1 className="text-3xl font-bold mb-6 text-center">Agregar Nueva Tarea</h1>
      <form onSubmit={formik.handleSubmit} className="space-y-4">
        <div>
          <label htmlFor="task" className="block text-lg font-medium mb-2">
            Tarea:
          </label>
          <input
            type="text"
            id="task"
            name="task"
            value={formik.values.task}
            onChange={formik.handleChange}
            className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
            placeholder="Escribe una tarea"
          />
        </div>
        <div>
          <label htmlFor="description" className="block text-lg font-medium mb-2">
            Descripción:
          </label>
          <textarea
            id="description"
            name="description"
            value={formik.values.description}
            onChange={formik.handleChange}
            className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
            placeholder="Escribe una descripción"
            rows="4"
          ></textarea>
        </div>
        <div>
          <label htmlFor="priority" className="block text-lg font-medium mb-2">
            Prioridad:
          </label>
          <select
            id="priority"
            name="priority"
            value={formik.values.priority}
            onChange={formik.handleChange}
            className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          >
            <option value="baja">Baja</option>
            <option value="media">Media</option>
            <option value="alta">Alta</option>
          </select>
        </div>
        <div className="flex items-center">
          <input
            type="checkbox"
            id="urgent"
            name="urgent"
            checked={formik.values.urgent}
            onChange={formik.handleChange}
            className="mr-2"
          />
          <label htmlFor="urgent" className="text-lg font-medium">
            Tarea Urgente
          </label>
        </div>
        <div>
          <label className="block text-lg font-medium mb-2">Categorías:</label>
          {categoryOptions.map((cat) => (
            <div key={cat} className="flex items-center">
              <input
                type="checkbox"
                id={cat}
                value={cat}
                checked={formik.values.categories.includes(cat)}
                onChange={() => handleCategoryChange(cat)}
                className="mr-2"
              />
              <label htmlFor={cat}>{cat}</label>
            </div>
          ))}
        </div>
        <button
          type="submit"
          className="w-full py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-200"
        >
          Agregar Tarea
        </button>
      </form>
    </div>
  );
}

export default AddTask;

Explicación Detallada:

• useFormik configura initialValues, onSubmit.

• handleChange maneja cambios en inputs automáticamente.

• Para checkboxes dinámicos, usamos setFieldValue para actualizar el array.

• onSubmit reemplaza handleSubmit para procesar valores.

11. React Hook Form (useForm y composeValidators)

¿Qué es React Hook Form? React Hook Form es una librería ligera para formularios, con useForm para manejo y composeValidators para validaciones compuestas.

Ejemplo Práctico Cambiamos a React Hook Form.

Modifica src/pages/AddTask.jsx: 

import { useForm } from 'react-hook-form';
import { composeValidators, required, minLength } from 'react-hook-form/dist/validators'; // Asumiendo import correcto, ajusta si es necesario
// ... resto

function AddTask() {
  const { register, handleSubmit, formState: { errors } } = useForm();
  const navigate = useNavigate();

  const onSubmit = (data) => {
    console.log('Tarea agregada:', data);
    Swal.fire({
      title: '¡Éxito!',

      icon: 'success',

      navigate('/tasks');

12. React Final Form

¿Qué es React Final Form? React Final Form es una librería para formularios basada en Final Form, con soporte para validaciones y rendimiento óptimo.

Ejemplo Práctico Cambiamos a React Final Form.

Modifica src/pages/AddTask.jsx: 

import { Form, Field } from 'react-final-form';
// ... resto

function AddTask() {
  const navigate = useNavigate();

  const onSubmit = (values) => {
    console.log('Tarea agregada:', values);
    Swal.fire({
      title: '¡Éxito!',
      text: 'Tarea agregada correctamente.',
      icon: 'success',
    }).then(() => {
      navigate('/tasks');
    });
  };

  const validate = (values) => {
    const errors = {};
    if (!values.task) errors.task = 'Requerida';
    if (values.task && values.task.length < 5) errors.task = 'Mínimo 5 caracteres';
    return errors;
  };

  return (
    <div className="container mx-auto p-4 max-w-md">
      <h1 className="text-3xl font-bold mb-6 text-center">Agregar Nueva Tarea</h1>
      <Form
        onSubmit={onSubmit}
        validate={validate}
        render={({ handleSubmit }) => (
          <form onSubmit={handleSubmit} className="space-y-4">
            <Field name="task">
              {({ input, meta }) => (
                <div>
                  <label htmlFor="task" className="block text-lg font-medium mb-2">
                    Tarea:
                  </label>
                  <input
                    {...input}
                    id="task"
                    className="w-full p-3 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
                    placeholder="Escribe una tarea"
                  />
                  {meta.error && meta.touched && <p className="text-red-500 text-sm mt-1">{meta.error}</p>}
                </div>
              )}
            </Field>
            // ... Similar para otros campos
            <button
              type="submit"
              className="w-full py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-200"
            >
              Agregar Tarea
            </button>
          </form>
        )}
      />
    </div>
  );
}

export default AddTask;

Explicación Detallada:

• <Form> maneja el formulario con onSubmit y validate.

• <Field> registra campos y muestra errores con meta.error y meta.touched.

• Validaciones son sincronas en validate.
