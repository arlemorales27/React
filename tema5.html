Introducción

Estas utilidades incluyen librerías y componentes para mejorar la funcionalidad, como manejo de fechas, spinners de carga, listas swipeables, webcam, gráficos, mapas, modales y carruseles de Bootstrap. Si no tienes el proyecto configurado, revisa las respuestas anteriores para crear task-app-tailwind con Vite, React Router, TailwindCSS y componentes básicos. Asegúrate de que el servidor esté corriendo con npm run dev.

1. Creación Componente de Página

¿Qué es la Creación de un Componente de Página? Un componente de página es un componente React que representa una vista completa, como una página en una SPA. Se integra con React Router para enrutamiento. Es reutilizable, maneja estado propio y se enfoca en la lógica de una sección específica (e.g., lista de tareas).

Explicación Detallada:

  • Usa hooks como useState, useEffect para estado y efectos.
  • Se monta en rutas específicas con <Route element={<ComponentePagina />} />.
  • En nuestra app, ya tenemos páginas como Tasks.jsx, pero crearemos una nueva página Dashboard.jsx para demostrar: un dashboard con resumen de tareas, contador y gráficos (integrando temas posteriores).
  • Ventajas: Modularidad, responsividad con TailwindCSS, y escalabilidad.

Paso 1: Crear el Componente Crea src/pages/Dashboard.jsx: 

import { useState, useEffect } from 'react';
import { Link } from 'react-router-dom';

function Dashboard() {
  const [totalTasks, setTotalTasks] = useState(0);
  const [completedTasks, setCompletedTasks] = useState(0);

  useEffect(() => {
    // Simular carga de datos
    setTimeout(() => {
      setTotalTasks(10);
      setCompletedTasks(4);
    }, 1000);
  }, []);

  return (
    <div className="container mx-auto p-4">
      <h1 className="text-3xl font-bold mb-6 text-center">Dashboard de Tareas</h1>
      <div className="grid grid-cols-1 md:grid-cols-2 gap-6 mb-6">
        <div className="bg-white p-6 rounded-lg shadow-md">
          <h2 className="text-xl font-semibold mb-2">Total de Tareas</h2>
          <p className="text-4xl font-bold text-blue-500">{totalTasks}</p>
        </div>
        <div className="bg-white p-6 rounded-lg shadow-md">
          <h2 className="text-xl font-semibold mb-2">Tareas Completadas</h2>
          <p className="text-4xl font-bold text-green-500">{completedTasks}</p>
        </div>
      </div>
      <div className="text-center">
        <Link to="/tasks" className="px-6 py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-200">
          Ver Todas las Tareas
        </Link>
      </div>
    </div>
  );
}

export default Dashboard;

2. Dayjs

¿Qué es Dayjs? Dayjs es una librería ligera para manejo de fechas y horas en JavaScript, similar a Moment.js pero más moderna y pequeña (2KB). Es inmutable, soporta plugins para internacionalización, y es ideal para formatear fechas, calcular diferencias, etc., en apps React.

Explicación Detallada:

  • Ventajas sobre Date nativo: API chainable, inmutable, soporte para locales.
  • Instalado ya. Usaremos en Dashboard.jsx para mostrar fechas de tareas creadas.
  • Plugins opcionales: e.g., dayjs/locale/es para español.

Paso 1: Importar y Usar Modifica src/pages/Dashboard.jsx: 

import dayjs from 'dayjs';
import 'dayjs/locale/es'; // Para español
dayjs.locale('es'); // Configurar locale global

function Dashboard() {
  const [totalTasks, setTotalTasks] = useState(0);
  const [completedTasks, setCompletedTasks] = useState(0);
  const [currentDate, setCurrentDate] = useState('');

  useEffect(() => {
    // Simular carga
    setTimeout(() => {
      setTotalTasks(10);
      setCompletedTasks(4);
      setCurrentDate(dayjs().format('DD MMMM YYYY')); // Formatear fecha actual
    }, 1000);
  }, []);

  return (
    <div className="container mx-auto p-4">
      <h1 className="text-3xl font-bold mb-6 text-center">Dashboard de Tareas</h1>
      <p className="text-center text-gray-600 mb-6">Fecha actual: {currentDate}</p>
      // ... resto del código ...
    </div>
  );
}

Paso 2: Ejemplo Avanzado - Diferencia de Fechas Agrega un ejemplo para calcular días hasta una fecha límite: 

// Dentro del useEffect:
const deadline = dayjs('2025-12-31');
const daysLeft = deadline.diff(dayjs(), 'day');
setDaysLeft(daysLeft);

// En el JSX:
<p className="text-center text-gray-600">Días hasta fin de año: {daysLeft}</p>

Explicación Detallada:

  • dayjs().format('DD MMMM YYYY') formatea la fecha actual en español.
  • diff calcula diferencias (e.g., días).
  • Inmutable: dayjs() no modifica la fecha original.
  • Locale: Configurado para español, pero puedes cambiarlo.

Prueba:

· Ve a /dashboard: Verifica la fecha formateada y el cálculo de días. Cambia el locale para probar internacionalización.

3. Moment

¿Qué es Moment? Moment.js es una librería popular para parsear, validar, manipular y formatear fechas. Es más pesada que Dayjs (66KB), pero ofrece más funcionalidades out-of-the-box, como soporte para timezones y duraciones. En 2025, se recomienda para proyectos legacy o complejos, pero Dayjs es preferido para nuevos.

Explicación Detallada:

  • API similar a Dayjs, pero con métodos como moment().fromNow().
  • Instalado ya. Usaremos en Tasks.jsx para mostrar fechas de creación de tareas.

Paso 1: Importar y Usar Modifica src/pages/Tasks.jsx: 

import moment from 'moment';
import 'moment/locale/es'; // Para español
moment.locale('es'); // Configurar locale

function Tasks() {
  const tasks = useLoaderData(); // Asumiendo datos con fecha
  // Agrega fecha a las tareas si no la tienen
  const tasksWithDates = tasks.map(task => ({
    ...task,
    createdAt: moment().subtract(Math.random() * 7, 'days').format('DD/MM/YYYY'), // Fecha aleatoria
  }));

  return (
    <div className="container mx-auto p-4">
      <h1 className="text-3xl font-bold mb-4">Mis Tareas</h1>
      <ul className="space-y-2">
        {tasksWithDates.map((task) => (
          <li key={task.id} className="bg-white p-3 rounded shadow-sm">
            <span>{task.text}</span>
            <p className="text-sm text-gray-500">Creada: {task.createdAt}</p>
            <p className="text-sm text-gray-500">Hace: {moment(task.createdAt, 'DD/MM/YYYY').fromNow()}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}

Paso 2: Ejemplo Avanzado - Duración 

// En el JSX:
<p className="text-sm text-gray-500">Duración estimada: {moment.duration(5, 'hours').humanize()}</p>

Explicación Detallada:

  • moment().format('DD/MM/YYYY') formatea fechas.
  • fromNow() muestra "hace 3 días".
  • Locale para español.
  • duration calcula duraciones humanas (e.g., "5 horas").

Prueba:

· Ve a /tasks: Verifica fechas formateadas y "hace X tiempo".

4. Creación de Spinner

¿Qué es la Creación de un Spinner? Un spinner es un componente de carga visual (e.g., círculo giratorio) para indicar que una operación está en progreso. Se crea con CSS puro o librerías, y se integra con estado de carga.

Explicación Detallada:

  • Usamos CSS para un spinner simple, responsivo.
  • Integramos en Tasks.jsx durante la carga de datos.

Paso 1: Crear el Componente Spinner Crea src/components/Spinner.jsx: 

import styled from '@emotion/styled';

const SpinnerContainer = styled.div`
  display: flex;
  justify-content: center;
  align-items: center;
  height: 200px;
`;

const Spinner = styled.div`
  width: 40px;
  height: 40px;
  border: 4px solid #f3f3f3;
  border-top: 4px solid #3498db;
  border-radius: 50%;
  animation: spin 1s linear infinite;
  @keyframes spin {
    0% { transform: rotate(0deg); }
    100% { transform: rotate(360deg); }
  }
`;

function Spinner() {
  return (
    <SpinnerContainer>
      <Spinner />
    </SpinnerContainer>
  );
}

export default Spinner;

Paso 2: Integrar en Tasks.jsx Modifica src/pages/Tasks.jsx: 

import Spinner from '../components/Spinner';
// ... resto

function Tasks() {
  const tasks = useLoaderData();
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    setTimeout(() => setLoading(false), 2000); // Simular carga
  }, []);

  if (loading) {
    return (
      <div className="container mx-auto p-4">
        <Spinner />
      </div>
    );
  }

  // ... resto del JSX ...
}

Explicación Detallada:

  • CSS con @keyframes crea la animación de rotación.
  • @emotion/styled encapsula estilos.
  • loading estado controla cuándo mostrar el spinner.
  • Responsivo: El spinner se centra en cualquier pantalla.

Prueba:

· Ve a /tasks: Ve el spinner durante 2 segundos, luego las tareas.

5. react-swipeable-list

¿Qué es react-swipeable-list? react-swipeable-list es una librería para crear listas swipeables (deslizables), útil para acciones como eliminar o editar elementos con gesto táctil en móviles.

Explicación Detallada:

  • Soporta swipe izquierda/derecha para revelar botones.
  • Instalado ya. Usaremos en Tasks.jsx para listas de tareas swipeables.

Paso 1: Importar y Usar Modifica src/pages/Tasks.jsx: 

import { SwipeableList, SwipeableListItem, SwipeAction } from 'react-swipeable-list';
import 'react-swipeable-list/drag-preview.css';
import 'react-swipeable-list/swipeable-list.css';

function Tasks() {
  const tasks = useLoaderData();
  // ... loading ...

  const swipeRightAction = (task) => (
    <SwipeAction
      onClick={() => console.log(`Completar ${task.text}`)}
      className="bg-green-500 text-white flex items-center justify-center"
    >
      Completar
    </SwipeAction>
  );

  const swipeLeftAction = (task) => (
    <SwipeAction
      onClick={() => console.log(`Eliminar ${task.text}`)}
      className="bg-red-500 text-white flex items-center justify-center"
    >
      Eliminar
    </SwipeAction>
  );

  return (
    <div className="container mx-auto p-4">
      <h1 className="text-3xl font-bold mb-4">Mis Tareas</h1>
      <SwipeableList>
        {tasks.map((task) => (
          <SwipeableListItem
            key={task.id}
            rightAction={swipeRightAction(task)}
            leftAction={swipeLeftAction(task)}
            swipeRightThreshold={30}
            swipeLeftThreshold={30}
          >
            <div className="bg-white p-3 rounded shadow-sm flex items-center justify-between">
              <span>{task.text}</span>
              <button className="px-2 py-1 bg-blue-500 text-white rounded">Editar</button>
            </div>
          </SwipeableListItem>
        ))}
      </SwipeableList>
    </div>
  );
}

Explicación Detallada:

  • <SwipeableList> envuelve la lista.
  • <SwipeableListItem> permite swipe, con rightAction y leftAction para botones.
  • threshold define sensibilidad al swipe (30px).
  • CSS importados para estilos por defecto.
  • Responsivo: Funciona en touch devices.

Prueba:

· Ve a /tasks: Desliza elementos izquierda/derecha para ver acciones.

6. Webcam

¿Qué es Webcam? react-webcam es una librería para acceder a la webcam del usuario en React, capturando video o fotos para apps como escáneres o avatares.

Explicación Detallada:

  • Requiere permiso del usuario.
  • Usaremos en AddTask.jsx para capturar una "foto" de la tarea (simulada como avatar).

Paso 1: Importar y Usar Modifica src/pages/AddTask.jsx: 

import Webcam from 'react-webcam';

function AddTask() {
  const webcamRef = useRef(null);
  const [imgSrc, setImgSrc] = useState(null);

  const capture = () => {
    const imageSrc = webcamRef.current.getScreenshot();
    setImgSrc(imageSrc);
  };

  return (
    <div className="container mx-auto p-4 max-w-md">
      <h1 className="text-3xl font-bold mb-6 text-center">Agregar Nueva Tarea</h1>
      <form className="space-y-4">
        // ... campos ...
        <div>
          <label className="block text-lg font-medium mb-2">Foto de Tarea (Webcam):</label>
          <Webcam
            audio={false}
            ref={webcamRef}
            screenshotFormat="image/jpeg"
            className="w-full rounded-lg"
          />
          <button
            type="button"
            onClick={capture}
            className="mt-2 px-4 py-2 bg-purple-500 text-white rounded hover:bg-purple-600"
          >
            Capturar Foto
          </button>
          {imgSrc && <img src={imgSrc} alt="Captura" className="mt-2 w-full rounded" />}
        </div>
        <button type="submit" className="w-full py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600">
          Agregar Tarea
        </button>
      </form>
    </div>
  );
}

Explicación Detallada:

  • <Webcam> captura video.
  • getScreenshot() toma foto.
  • useRef referencia el componente.
  • Permiso: El navegador pedirá acceso a cámara.
  • Responsivo: Se adapta a la pantalla.

Prueba:

· Ve a /add-task: Permite cámara, captura foto y ve la imagen.

7. Gráficos con react-chartjs-2

¿Qué es react-chartjs-2? react-chartjs-2 es un wrapper de Chart.js para React, permitiendo gráficos interactivos (barras, líneas, etc.) en componentes.

Explicación Detallada:

  • Instalado ya. Usaremos en Dashboard.jsx para gráfico de tareas completadas.

Paso 1: Importar y Usar Modifica src/pages/Dashboard.jsx: 

import { Bar } from 'react-chartjs-2';
import {
  Chart as ChartJS,
  CategoryScale,
  LinearScale,
  BarElement,
  Title,
  Tooltip,
  Legend,
} from 'chart.js';

ChartJS.register(CategoryScale, LinearScale, BarElement, Title, Tooltip, Legend);

function Dashboard() {
  // ... estado ...

  const chartData = {
    labels: ['Completadas', 'Pendientes'],
    datasets: [
      {
        label: 'Tareas',
        data: [completedTasks, totalTasks - completedTasks],

9. Ventana Modal Bootstrap

¿Qué es Ventana Modal Bootstrap? Bootstrap modales son diálogos overlay para confirmaciones o formularios. Usamos react-bootstrap para integración en React.

Explicación Detallada:

  • Instalado ya. Usaremos en Tasks.jsx para confirmar eliminación.

Paso 1: Importar y Usar Modifica src/pages/Tasks.jsx: 

import { Modal, Button } from 'react-bootstrap';

function Tasks() {
  const [showModal, setShowModal] = useState(false);
  const [taskToDelete, setTaskToDelete] = useState(null);

  const handleDelete = (task) => {
    setTaskToDelete(task);
    setShowModal(true);
  };

  const confirmDelete = () => {
    console.log('Eliminar tarea:', taskToDelete.text);
    setShowModal(false);
  };

  return (
    <div className="container mx-auto p-4">
      // ... lista ...
      <Button onClick={() => handleDelete(task)} variant="danger">
        Eliminar
      </Button>

      <Modal show={showModal} onHide={() => setShowModal(false)}>
        <Modal.Header closeButton>
          <Modal.Title>Confirmar Eliminación</Modal.Title>
        </Modal.Header>
        <Modal.Body>¿Eliminar "{taskToDelete?.text}"?</Modal.Body>
        <Modal.Footer>
          <Button variant="secondary" onClick={() => setShowModal(false)}>
            Cancelar
          </Button>
          <Button variant="danger" onClick={confirmDelete}>
            Eliminar
          </Button>
        </Modal.Footer>
      </Modal>
    </div>
  );
}

Explicación Detallada:

  • <Modal> maneja visibilidad con show.
  • onHide cierra el modal.
  • Responsivo: Bootstrap lo hace mobile-friendly.

Prueba:

· En /tasks, haz clic en eliminar: Ve el modal de confirmación.
