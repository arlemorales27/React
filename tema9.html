Concepto General

Ant Design es una biblioteca de UI para React, pero aquí implementamos sus componentes en vanilla JS con HTML5 y CSS3. Usamos clases CSS personalizadas para replicar estilos (colores, espaciados, animaciones como fadeIn) y JS para interacciones dinámicas. Todo responsivo con media queries. Cada subtema explica el concepto, el código y 2 ejemplos.

Instalación Ant Design

Concepto: En vanilla JS, "instalamos" enlazando CSS/JS de Ant Design o recreando estilos. Aquí usamos estilos custom en <style> para simplicidad, sin dependencias externas.

<!-- En <head> -->
<style>
  .ant-btn { /* Estilos Ant-like */ }
</style>
<!-- JS para interacciones -->
<script>
  function initAnt() { /* Inicializar componentes */ }
</script>

Explicación: Las clases CSS como .ant-btn replican el diseño de Ant. JS maneja eventos sin frameworks.

Validación formulario | rules

Concepto: Reglas de validación en JS (e.g., required, pattern). Muestra errores en submit.

function validate(form) {
  const email = form.querySelector('[name="email"]').value;
  if (!email.includes('@')) { alert('Email inválido'); return false; }
  return true;
}
<form onsubmit="return validate(this)">...

Explicación: Función chequea fields; return false bloquea submit.

Table, columns y dataSource

Concepto: Tabla dinámica: columns definen headers, dataSource es array de rows.

const columns = [{ title: 'Nombre', dataIndex: 'name' }];

let table = '';
dataSource.forEach(row => { table += `<tr><td>${row.name}</td></tr>`; });
document.querySelector('.ant-table tbody').innerHTML = table;

Explicación: JS loop genera rows desde dataSource.
